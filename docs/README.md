# SPECTRA Documentation

This directory contains the SPECTRA documentation built with [Docusaurus](https://docusaurus.io/), a modern static website generator.

## Structure

```
docs/
├── docs/              # Source markdown files (Docusaurus content)
├── src/               # React components and custom CSS
├── static/            # Static assets (images, etc.)
├── html/              # Built HTML output (generated by `npm run build`)
├── defunct/           # Archived old markdown documentation
├── docusaurus.config.js  # Docusaurus configuration
├── sidebars.js        # Sidebar navigation structure
└── package.json       # Node.js dependencies
```

## Quick Start

### Prerequisites

- Node.js 18.0 or higher
- npm or yarn

### Installation

```bash
cd docs
npm install
```

### Development Server

Start a local development server with hot-reload:

```bash
npm start
```

The documentation will be available at `http://localhost:3000`

### Build HTML Documentation

Build the static HTML files:

```bash
npm run build
```

The HTML output will be generated in `docs/html/` directory.

### Serve Built Documentation

Serve the built HTML files locally:

```bash
npm run serve
```

The built documentation will be available at `http://localhost:3000`

## Entry Point

The root `index.html` in the SPECTRA directory automatically redirects to `docs/html/index.html` when the documentation is built.

## Documentation Structure

- **Getting Started**: Installation, quick start, and configuration
- **User Guides**: TUI usage, forwarding, shunt mode, launcher, web interface
- **Deployment**: Docker and systemd deployment guides
- **Features**: Advanced features, threat scoring, AI intelligence, vector database
- **API Reference**: CLI commands and API documentation
- **Reference**: Database schema, security, architecture, changelog

## Updating Documentation

1. Edit markdown files in `docs/docs/`
2. Update `sidebars.js` if adding new sections
3. Test locally with `npm start`
4. Build with `npm run build` when ready

## Deployment

For GitHub Pages deployment:

```bash
npm run deploy
```

This will build and deploy to the `gh-pages` branch automatically.

## Legacy Documentation

Old markdown files have been moved to `defunct/` for historical reference. They are no longer actively maintained.
